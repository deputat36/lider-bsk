# Усиление публичной формы заявок

Дата: 2026-06-23.

## Текущее состояние

Форма сайта отправляет заявку в Edge Function `leader-public-lead`. Клиентский код собирает имя, телефон, услугу, страницу, UTM-метки и дополнительные поля заявки.

Edge Function `leader-public-lead` активна как публичная функция без JWT. Это ожидаемо для формы сайта: функция сама валидирует payload, нормализует телефон, проверяет honeypot и пишет аудит.

## Что уже сделано в коде формы

- Генерируется `request_id` при отправке.
- `request_id` передаётся в payload.
- Передаются `submitted_at`, `page_path`, `user_agent` и технические данные источника.
- Повторная отправка той же формы блокируется, пока текущий запрос не завершён.

## Что уже сделано на сервере

- `request_id` сохраняется в `leader_leads`.
- Для `leader_leads.request_id` добавлена уникальность.
- Повторная отправка уже принятого `request_id` не создаёт дубль.
- Телефон нормализуется и сохраняется в `phone_normalized`.
- Создан аудит публичных отправок в `leader_public_lead_audit`.
- Ошибка записи аудита не блокирует получение основной заявки.

## Права доступа

Таблица основных заявок:

`public.leader_leads`

Минимальная модель GRANT после усиления 2026-06-23:

- `anon`: только `INSERT`;
- `authenticated`: `SELECT`, `INSERT`, `UPDATE`, `DELETE`;
- `TRUNCATE`, `REFERENCES`, `TRIGGER` отозваны у публичных ролей;
- `SELECT`, `UPDATE`, `DELETE` отозваны у `anon`.

Миграция:

`supabase/migrations/20260623_tighten_leader_leads_grants.sql`

Таблица аудита:

`public.leader_public_lead_audit`

Минимальная модель GRANT:

- `anon`: только `INSERT`;
- `authenticated`: только `SELECT`.

Миграция:

`supabase/migrations/20260623_tighten_leader_public_lead_audit_grants.sql`

## Проверка

Проверено через Supabase:

- `leader-public-lead` активна, версия 6, `verify_jwt=false`;
- live-исходник функции совпадает с ожидаемой логикой в репозитории;
- RLS включён на `leader_leads` и `leader_public_lead_audit`;
- rollback smoke-test вставки в `leader_leads` под ролью `anon` прошёл;
- rollback smoke-test вставки в `leader_public_lead_audit` под ролью `anon` прошёл;
- постоянные тестовые записи не создавались.

## Что ещё нужно проверить вручную

1. Открыть публичный сайт.
2. Отправить тестовую заявку.
3. Открыть CRM v4.
4. Открыть раздел `Аудит заявок`.
5. Нажать `Обновить аудит`.
6. Проверить, что появилась запись `Принято`.
7. Проверить, что основная заявка появилась в списке заявок CRM.

## Возможная следующая доработка

Если в аудит начнут попадать массовые мусорные отправки, добавить мягкий rate-limit по IP/User-Agent на уровне Edge Function или отдельной таблицы счётчиков.
