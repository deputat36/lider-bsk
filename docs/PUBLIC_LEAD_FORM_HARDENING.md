# Усиление публичной формы заявок

Дата: 2026-06-21.

## Текущее состояние

Форма сайта отправляет заявку в Edge Function `leader-public-lead`. Клиентский код собирает имя, телефон, услугу, страницу, UTM-метки и дополнительные поля заявки.

## Что нужно доработать в коде формы

1. Генерировать `request_id` при каждой отправке.
2. Передавать `request_id` в payload.
3. Передавать `submitted_at`, `page_path`, `user_agent`.
4. Блокировать повторную отправку той же формы, пока текущий запрос не завершён.
5. При сетевой ошибке не создавать новый `request_id` автоматически до следующего осознанного клика пользователя.

## Что нужно доработать на сервере

1. Принимать только заявки с `request_id`.
2. Хранить `request_id` в `leader_leads`.
3. Возвращать успех при повторе уже принятого `request_id`, не создавая дубль.
4. Нормализовать телефон и сохранять `phone_normalized`.
5. Добавить rate limit по IP/User-Agent, если Edge Function получает много мусорных запросов.

## Почему это важно

Даже если пользователь дважды нажмёт кнопку, обновит страницу или запрос повторится из-за нестабильной сети, CRM должна получить одну заявку, а не несколько дублей.
