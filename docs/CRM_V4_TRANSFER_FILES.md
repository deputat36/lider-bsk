Дата: 2026-06-22.

Рабочая ссылка для проверки:

`https://deputat36.github.io/lidercalculator/app-v4.html`

Перенос выполняется пакетами. В основной репозиторий уже перенесён рабочий путь внутри карточки заявки:

## Важные решения

- `lead-card.js` исправлен: возврат из карточки к списку больше не запускает петлю route-change.
- `crm/v4/index.html` подключает только реально перенесённые рабочие модули.
- `management-dashboard-v3.js` перенесён с адаптацией под уже существующие вкладки: `leads`, `contact_control`, `order_control`, `production`, `finance_control`.
- `management-dashboard-v3.js` не ссылается на ещё неперенесённые `workdesk` и `production_control`.
- `orders.js` в основном репозитории адаптирован: создание заказа идёт через `leader-crm-leads` → `create_order_from_offer`, а не через старый клиентский сценарий `create_order` с ручным связыванием.
- `orders-fast-loader-v1.js` создаёт быстрый список заказов, читает `leader_orders`, открывает `order-card-v1.js` через `data-open-order` и может вернуть пользователя к связанной заявке.
- `order-card-v1.js` сделан как новая базовая карточка заказа, потому отдельного исходного файла `order-card-v1.js` во временной CRM не было.
- `order-control-v2.js` перенесён с адаптацией: при открытии раздела он скрывает базовые блоки заявок и работает как управляемая вкладка `order_control`.
- `finance-control-v2.js` перенесён с адаптацией: при открытии раздела он скрывает базовые блоки заявок и работает как управляемая вкладка `finance_control`.
- `production-board-v3.js` перенесён с адаптацией: при открытии раздела он скрывает базовые блоки заявок и работает как управляемая вкладка `production`.
- `production-board-v3.js` теперь открывает `production-job-card-v2.js` и `installation-job-card-v2.js` из карточек заданий, а также даёт печать листов и переход в связанный заказ.
- `production-alerts-v1.js` адаптирован под лёгкую доску `production-board-v3.js` и обновляется после фактического рендера доски.
- `crm-v4-tabs-lite.js` управляет вкладками `management_dashboard`, `orders`, `order_control`, `finance_control` и `production` вместе с другими управляемыми разделами.
- `calculations.js` остаётся в репозитории как перенесённый базовый модуль, но `index.html` использует расширенную связку расчётов.

- `crm/v4/index.html` подключает расширенные расчётные модули и больше не подключает `calculations.js`.

Проверено через Supabase connector:

- `leader-public-lead` активна, версия 6, `verify_jwt=false`;
- `leader-crm-leads` активна, версия 7, `verify_jwt=true`;
- `leader-crm-orders` активна, версия 2, `verify_jwt=true`;
- `leader-crm-leads` содержит действие `create_order_from_offer`.

Ограничение проверки:

- HTTP-запрос из текущего окружения к `https://deputat36.github.io/lider-bsk/crm/v4/` возвращает `403`;
- raw GitHub download из текущего окружения через терминал тоже возвращает `403`, поэтому локальный `node --check` через curl не выполнен;
- нужна ручная проверка в обычном браузере владельца после Ctrl + F5.

## Рекомендуемый порядок проверки

1. Открыть `crm/v4/` в основном репозитории через GitHub Pages.
2. Проверить вход и выход.
3. Открыть `Дашборд`.
4. Проверить счётчики дашборда и быстрые переходы в заявки, контроль контактов, контроль заказов, производство и финансы.
5. Проверить загрузку списка заявок.
6. Открыть карточку заявки.
7. Проверить возврат `Назад к списку`.
8. Добавить потребность.
9. Создать типовой или нестандартный расчёт.
10. Создать КП из сохранённого расчёта.
11. Открыть карточку КП.
12. Проверить копирование короткого и полного текста.
13. Открыть PDF строгий и PDF презентацию.
14. Отметить КП как согласованное.
15. Создать заказ из согласованного КП.
16. Проверить, что блок `Заказ` появился в карточке заявки.
17. Проверить, что повторное создание заказа заблокировано.
18. Открыть вкладку `Заказы`.
19. Проверить, что список заказов загрузился и не показывает рядом старые блоки заявок.
20. Открыть `Карточка заказа` из списка заказов.
21. Проверить суммы, оплату, статусы, срок и позиции заказа.
22. Из карточки заказа открыть связанную заявку, если у заказа есть `lead_id`.
23. Открыть `Контроль заказов`.
24. Открыть `Финансы`.
25. Открыть `Производство`.
26. Проверить счётчики: производственных, производство открыто, монтажей, монтаж открыт, просрочено.
27. Проверить переключатель `Производство / Монтаж`.
28. Открыть карточку производственного задания, сохранить тестовое изменение и проверить печать листа.
29. Открыть карточку монтажного задания, сохранить тестовое изменение и проверить печать листа.
30. Открыть связанный заказ из производственного или монтажного задания.
31. Открыть `Контроль контактов`.
32. Открыть `Аудит заявок`.
33. Нажать `Проверить CRM` в диагностическом блоке.
34. Проверить консоль браузера: не должно быть 404 по CSS/JS из `assets/v4/`.

## Что ещё не перенесено

- расширенное меню рабочей CRM;
- при необходимости отдельный рабочий стол менеджера.

## Рекомендуемый порядок следующего переноса

1. Перенести расширенное меню и, если нужно, рабочий стол менеджера.
2. После переноса очередного пакета обновлять `crm/v4/index.html` только теми разделами, которые действительно работают.
3. Только после проверки переключать основной адрес CRM v4.

## Ограничения

- Рабочую версию в `lidercalculator` не удалять до завершения проверки.
- Старую CRM v2 не удалять до отдельного подтверждения.
- Таблицы `nav_*` не использовать.
- Крупные JS-файлы лучше переносить логическими пакетами, а не случайной частичной вставкой.
