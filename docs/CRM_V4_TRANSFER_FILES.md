# CRM v4 transfer notes

Дата: 2026-06-21.

Рабочая ссылка для проверки:

`https://deputat36.github.io/lidercalculator/app-v4.html`

Основная цель: перенести рабочую CRM v4 из `deputat36/lidercalculator` в `deputat36/lider-bsk/crm/v4/`.

## Текущее состояние

Перенос должен выполняться пакетами, потому что одной HTML-страницы недостаточно. Нужно переносить связанные CSS и JS-модули вместе с зависимостями.

В основной репозиторий уже перенесён критичный модуль входа:

- `crm/v4/assets/v4/auth.js`.

Попытка перенести `functions-client.js` через веб-инструмент была заблокирована системой безопасности инструмента из-за кода авторизованного вызова Edge Functions. Этот файл лучше переносить локальным пакетом или через обычный git commit, чтобы не ломать код и не обходить защитные ограничения.

Крупный файл `calculations.js` не переносился через веб-инструмент, потому что ответ GitHub по нему обрезается и есть риск создать неполный JS-файл.

## Рекомендуемый порядок переноса

1. Сначала перенести базовые зависимости:
   - `config.js`;
   - `api.js`;
   - `state.js`;
   - `ui.js`;
   - `supabase-client.js`;
   - `functions-client.js`;
   - `auth.js`;
   - `router.js`.

2. Затем перенести заявки и карточку заявки:
   - `leads.js`;
   - `lead-card.js`;
   - `needs.js`;
   - `calculations.js` или актуальные модульные файлы расчётов;
   - файлы сохранённых расчётов.

3. Затем перенести КП и заказы:
   - `offer-card-v1.js`;
   - `offer-print.js`;
   - `offer-print-brand-v4.js`;
   - `offer-order-create-v1.js`;
   - заказы и карточку заказа.

4. Затем перенести управленческие разделы:
   - рабочий стол;
   - дашборд;
   - финансы;
   - производство;
   - монтаж;
   - аудит заявок;
   - self-check.

5. После этого обновить `crm/v4/index.html`.

6. Только после проверки переключать основной адрес CRM v4.

## Ограничения

- Рабочую версию в `lidercalculator` не удалять до завершения проверки.
- Старую CRM v2 не удалять до отдельного подтверждения.
- Таблицы `nav_*` не использовать.
- Крупные JS-файлы лучше переносить локальным commit-пакетом, а не точечной вставкой через веб-инструмент.
