# Проверка самодиагностики CRM v4

Дата: 2026-06-22.

CRM v4:

`https://deputat36.github.io/lider-bsk/crm/v4/`

## Зачем нужна проверка

Самодиагностика помогает быстро понять, что пользователь действительно вошёл в CRM, профиль доступа загрузился, роль подходит для работы и основные разделы меню подключены без дублей.

Это особенно важно после закрытия прямого RPC-доступа к `leader_ensure_profile`: CRM должна использовать свежий `auth.js`, который готовит профиль через Edge Function `leader-crm-leads`.

## Что изменено

В `crm-ui-selfcheck-v1.js` добавлена проверка:

- статуса входа;
- email текущего пользователя;
- роли профиля;
- активности профиля;
- текущего раздела CRM;
- URL страницы;
- наличия основных разделов CRM в меню.

В `crm/v4/index.html` обновлены cache-buster версии:

- `auth.js?v=20260622-1`;
- `site-cache-note-v1.js?v=20260622-2`.

В `site-cache-note-v1.js` обновлена версия импорта:

- `crm-ui-selfcheck-v1.js?v=20260622-2`.

## Автоматическая защита

GitHub Actions `Static checks` теперь дополнительно проверяет CRM v4:

- наличие `crm/v4/index.html`;
- наличие `auth.js`, `functions-client.js`, `site-cache-note-v1.js`, `crm-ui-selfcheck-v1.js`;
- актуальные версии подключения `auth.js`, `site-cache-note-v1.js`, `crm-ui-selfcheck-v1.js`;
- что `auth.js` использует `leader-crm-leads` action `ensure_profile`;
- что `auth.js` не вызывает `leader_ensure_profile` напрямую.

Это не заменяет ручной вход в CRM, но ловит типовые ошибки кэша и подключения файлов до ручной проверки.

## Как проверить владельцу или администратору

1. Открыть CRM v4.
2. Нажать `Ctrl + F5`.
3. Войти под своим пользователем.
4. Открыть блок `Проверка загруженных разделов и доступа CRM`.
5. Проверить блок `Доступ`:
   - статус входа без ошибки;
   - email соответствует текущему пользователю;
   - роль одна из `owner`, `admin`, `manager`;
   - профиль: `Активен`;
   - текущий раздел заполнен;
   - URL соответствует CRM v4.
6. Проверить блок `Разделы`:
   - у каждого раздела статус `OK`;
   - нет строк `нет кнопки`;
   - нет строк `дублей`.

## Проверка для тестировщика `kvmbsk@yandex.ru`

Ожидаемые значения:

- email: `kvmbsk@yandex.ru`;
- роль: `admin`;
- профиль: `Активен`.

Если хотя бы одно значение отличается, тестировщик должен прислать скриншот блока самодиагностики и указать, на каком шаге возникла проблема.

## Что считать проблемой

- CRM просит вход после успешного логина;
- роль пустая или не `admin` для `kvmbsk@yandex.ru`;
- профиль не `Активен`;
- самодиагностика не появляется после `Ctrl + F5`;
- в разделах есть `нет кнопки`;
- в разделах есть `дублей`;
- заявки, аудит, финансы или дашборд не открываются из меню.

## Проверено в Supabase

На 2026-06-22 активные профили CRM есть только в разрешённых ролях:

- `admin`: 1;
- `manager`: 1;
- `owner`: 2.

Активные Edge Functions проекта:

- `leader-public-lead` v6, публичная, без JWT;
- `leader-crm-leads` v8, JWT включён;
- `leader-crm-orders` v2, JWT включён.

Дополнительно проверено: в `public` нет `SECURITY DEFINER` функций с префиксом `leader_`, которые напрямую исполняются ролями `anon` или `authenticated`.
