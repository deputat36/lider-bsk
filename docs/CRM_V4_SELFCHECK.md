# Проверка самодиагностики CRM v4

Дата: 2026-06-27.

CRM v4:

`https://deputat36.github.io/lider-bsk/crm/v4/`

Прямая проверка вкладки `Доступ`:

`https://deputat36.github.io/lider-bsk/crm/v4/?tab=user_admin`

## Зачем нужна проверка

Самодиагностика помогает быстро понять, что пользователь действительно вошёл в CRM, профиль доступа загрузился, роль подходит для работы и основные разделы меню подключены без дублей.

Это особенно важно после закрытия прямого RPC-доступа к `leader_ensure_profile`: CRM должна использовать свежий `auth.js`, который готовит профиль через Edge Function `leader-crm-leads`, а управление доступом должно идти через вкладку `Доступ` и invite-flow.

## Что проверяется

В `crm-ui-selfcheck-v1.js` проверяется:

- контур и ключ сессии;
- статус входа;
- email текущего пользователя;
- роль профиля;
- активность профиля;
- текущий раздел CRM;
- URL страницы;
- наличие основных разделов CRM в меню;
- маршрут `Доступ`;
- наличие `window.v4SetTab`;
- наличие секции `userAdminSection`;
- версия доступа `20260627-access-route-1`;
- build marker `CRM build: 20260627-access-route-1`.

## Актуальные cache-buster версии

В `crm/v4/index.html` актуальны:

- `crm-v4-tabs-lite.js?v=20260627-access-route-1`;
- `site-cache-note-v1.js?v=20260627-access-route-1`;
- `auth.js?v=20260627-access-3`.

В `site-cache-note-v1.js` актуален импорт:

- `crm-ui-selfcheck-v1.js?v=20260627-access-route-1`.

## Автоматическая защита

GitHub Actions проверяют CRM v4:

- `Static checks` контролирует базовые CRM asset markers;
- `CRM access admin check` контролирует access/admin markers;
- `CRM access quick link check` контролирует кнопку `Открыть доступ CRM`, direct route, runbook, status, issue template, checklist/report и access/onboarding docs.

Это не заменяет ручной вход в CRM, но ловит типовые ошибки кэша, подключения файлов и устаревших инструкций до ручной проверки.

## Как проверить владельцу или администратору

1. Открыть CRM v4.
2. Нажать `Ctrl + F5`.
3. Войти под своим пользователем.
4. Открыть блок `Проверка загруженных разделов и доступа CRM`.
5. Проверить блок `Контур и сессия`:
   - `Контур: Основной`;
   - ключ сессии `leader_crm_v4_main_session`;
   - старый общий ключ очищен.
6. Проверить блок `Доступ`:
   - статус входа без ошибки;
   - email соответствует текущему пользователю;
   - роль одна из `owner`, `admin`, `manager`;
   - профиль: `Активен`;
   - текущий раздел заполнен;
   - URL соответствует CRM v4.
7. Проверить блок `Маршрут Доступ`:
   - прямой маршрут `?tab=user_admin` открыт или доступен;
   - роутер вкладок `OK`;
   - секция `Доступ` создана;
   - версия доступа `20260627-access-route-1`;
   - `Build marker` виден после свежей загрузки.
8. Открыть `Доступ` одним из способов:
   - прямой URL `https://deputat36.github.io/lider-bsk/crm/v4/?tab=user_admin`;
   - кнопка `Открыть доступ CRM` в карточке `CRM готова`;
   - верхняя вкладка `Доступ`.
9. Для `owner/admin` проверить, что видны пользователи CRM и приглашения.
10. Для пользователя без роли `owner/admin` проверить, что виден role-message вместо admin controls.
11. Проверить блок `Разделы`:
   - у каждого раздела статус `OK`;
   - нет строк `нет кнопки`;
   - нет строк `дублей`.

## Проверка для тестировщика `kvmbsk@yandex.ru`

Ожидаемые значения:

- email: `kvmbsk@yandex.ru`;
- роль: `admin`;
- профиль: `Активен`;
- вкладка `Доступ` открывается;
- для admin видны пользователи CRM и приглашения.

Если хотя бы одно значение отличается, тестировщик должен прислать скриншот блока самодиагностики и указать, на каком шаге возникла проблема.

## Что считать проблемой

- CRM просит вход после успешного логина;
- роль пустая или не `admin` для `kvmbsk@yandex.ru`;
- профиль не `Активен`;
- самодиагностика не появляется после `Ctrl + F5`;
- `CRM build: 20260627-access-route-1` не виден после свежей загрузки;
- в разделах есть `нет кнопки`;
- в разделах есть `дублей`;
- вкладка `Доступ` отсутствует;
- кнопка `Открыть доступ CRM` отсутствует;
- прямой URL `?tab=user_admin` не открывает `Доступ`;
- для owner/admin не видны пользователи CRM или приглашения;
- заявки, аудит, финансы или дашборд не открываются из меню.

## Проверено в Supabase

На 2026-06-27 активные профили CRM есть в разрешённых ролях:

- `admin`: 1;
- `manager`: 1;
- `owner`: 2.

Активные Edge Functions проекта:

- `leader-public-lead` v9, публичная, без JWT;
- `leader-crm-leads` v12, JWT включён;
- `leader-crm-orders` v2, JWT включён.

Дополнительно проверено:

- `leader_user_profiles` и `leader_user_invites` имеют RLS;
- grants для таблиц доступа: только `authenticated:SELECT`, `authenticated:INSERT`, `authenticated:UPDATE`;
- invite/profile triggers присутствуют;
- в `public` нет `SECURITY DEFINER` функций с префиксом `leader_`, которые напрямую исполняются ролями `anon`, `authenticated` или `public`.
