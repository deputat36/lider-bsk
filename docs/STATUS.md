## Supabase и публичные заявки

Проверено через Supabase:

- `leader-public-lead` активна, версия 6, публичная, без JWT;
- `leader-crm-leads` активна, версия 7, JWT включён;
- `leader-crm-orders` активна, версия 2, JWT включён.

Выполнено:

- в `leader_leads` добавлены `request_id`, `phone_normalized`, `source_page_path`, `submitted_at`, `client_user_agent`;
- добавлена уникальность `leader_leads.request_id`;
- добавлен индекс для поиска дублей по `phone_normalized` и дате создания;
- отправка заявки использует `on_conflict=request_id` и игнорирует повторную вставку дубля;
- создана таблица `leader_public_lead_audit`;
- RLS для `leader_public_lead_audit` включён;
- добавлены индексы аудита по дате, `request_id` и нормализованному телефону;
- чтение аудита доступно активным сотрудникам ролей `owner`, `admin`, `manager`;
- запись аудита разрешена публичному anon-контуру Edge Function;
- `leader-public-lead` пишет аудит для событий `accepted`, `suspicious`, `rejected`, `error`;
- ошибка записи аудита не блокирует получение основной заявки.

## Перенос CRM v4 в основной репозиторий

Перенос идёт в каталог:

`crm/v4/`

В основной репозиторий уже перенесены или добавлены:

- базовые CSS и JS CRM v4;
- авторизация, состояние, роутер, UI-хелперы и Supabase-клиент;
- список заявок, карточка заявки, ручное создание заявки;
- потребности клиента;
- контроль контактов;
- аудит публичных заявок;
- сохранённые, типовые и нестандартные расчёты;
- коммерческие предложения;
- карточка КП;
- печать/PDF КП с HTML/CSS-логотипом;
- создание заказа из согласованного КП через `leader-crm-leads`.

Критичные исправления:

- `crm/v4/index.html` ссылался на модули и CSS-файлы, которых не было в `lider-bsk`; недостающие базовые файлы добавлены;
- `lead-card.js` исправлен: возврат из карточки к списку больше не должен запускать петлю route-change;
- базовый `calculations.js` остаётся в репозитории как перенесённый модуль, но в `index.html` не подключается;
- `crm/v4/index.html` переключён на расширенную связку расчётов: `calculations-saved-tools-v2.js`, `calculations-standard.js`, `calculations-advanced.js`;
- `crm/v4/index.html` подключает пакет КП: `offers.css`, `offers.js`, `offer-card-v1.js`, `offer-print.js`, `offer-order-create-v1.js`.

`crm/v4/index.html` сейчас поддерживает:

- вход и выход;
- загрузку списка заявок;
- открытие карточки заявки;
- возврат из карточки к списку;
- смену статуса заявки;
- сохранение следующего контакта;
- ручное создание заявки;
- добавление и архивирование потребностей;
- просмотр сохранённых расчётов и состава расчёта;
- создание типового расчёта;
- создание нестандартного расчёта;
- формирование КП из сохранённого расчёта;
- копирование короткого и полного текста КП;
- смену статуса КП;
- карточку КП;
- печать/PDF КП;
- создание заказа из согласованного КП;
- контроль контактов;
- аудит публичных заявок;
- диагностический блок CRM.

Проверено через GitHub connector:

- подключённые CSS/JS существуют в `crm/v4/assets/v4/`;
- `lead-card.js` содержит исправленный `showLeadList(updateUrl = false)`;
- расчётные модули существуют и импортируют штатные `supabase-client`, `api`, `state`, `ui`;
- модули КП существуют и подключены в `index.html`;
- `offer-order-create-v1.js` использует `invokeLeaderFunction('leader-crm-leads', ...)` для серверного создания заказа.

Ограничение проверки:

- HTTP-запрос из текущего окружения к `https://deputat36.github.io/lider-bsk/crm/v4/` возвращает `403`;
- raw GitHub download из текущего окружения тоже возвращает `403`, поэтому локальный `node --check` через curl не выполнен;
- браузерная проверка в обычном браузере ещё нужна.

Важно:

- рабочая CRM v4 пока остаётся в `lidercalculator`;
- разделы заказов, финансы, производство, монтаж и дашборд ещё не перенесены в основной репозиторий полностью;
- создание заказа из согласованного КП уже перенесено, но отдельная карточка заказа и списки заказов ещё впереди.

## Коммерческие предложения и заказы

В основном репозитории уже перенесено:

- КП создаётся из сохранённого расчёта;
- клиентский текст КП не раскрывает внутренние финансовые данные;
- печатный/PDF-модуль КП использует встроенный HTML/CSS-логотип без внешней картинки;
- заказ создаётся только из согласованного КП;
- создание заказа идёт через серверное действие `create_order_from_offer` в `leader-crm-leads`;
- есть защита от повторного создания заказа на стороне UI и серверного действия.

Ещё нужно перенести:

- список заказов;
- отдельную карточку заказа;
- финансовый контроль;
- производство и монтаж;
- управленческий дашборд.
