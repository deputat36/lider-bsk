# Статус проекта РА «Лидер»

Дата обновления: 2026-06-21.

## Уже работает

### Публичный сайт

Сайт РА «Лидер» находится в основном репозитории `deputat36/lider-bsk`.

Проверено после аудита:

- `robots.txt` существует и оформлен построчно;
- `sitemap.xml` существует и имеет XML-структуру;
- заявки с сайта отправляются в Supabase Edge Function проекта CRM.

### Документация

Созданы и добавлены в основной репозиторий:

- `AGENTS.md`;
- `docs/PROJECT.md`;
- `docs/STATUS.md`;
- `docs/DECISIONS.md`;
- `docs/AUDIT_IMPROVEMENTS.md`.

### Автоматические проверки

Добавлен первый контур GitHub Actions:

- проверка `robots.txt`;
- проверка XML-структуры `sitemap.xml`;
- базовый поиск случайно закоммиченных секретных ключей.

### CRM v4

CRM v4 временно размещена в `deputat36/lidercalculator`.

Рабочий путь CRM v4:

1. Вход.
2. Список заявок.
3. Карточка заявки.
4. Потребности клиента.
5. Сохранённые расчёты.
6. Коммерческие предложения.
7. Создание заказа из согласованного КП.

### Перенос CRM v4 в основной репозиторий

Начат перенос в каталог:

`crm/v4/`

Уже перенесены:

- `crm/v4/index.html`;
- базовые CSS и JS-файлы ядра;
- конфигурация CRM v4;
- helper-файлы API, UI, роутера и состояния.

Полный перенос модулей авторизации, заявок, потребностей, расчётов, КП и заказов нужно завершить отдельным безопасным шагом, чтобы не перенести неполные файлы и не нарушить работу текущей версии.

### Коммерческие предложения

Реализовано во временной CRM v4:

- формирование КП по сохранённому расчёту;
- короткий текст для мессенджеров;
- подробное предложение клиенту;
- копирование текстов;
- статусы КП;
- история событий;
- клиентский текст без внутренних финансовых данных.

### Создание заказа

Реализовано во временной CRM v4:

- создание заказа только из согласованного КП;
- перенос клиента, позиций и сумм;
- связь заявки, потребности, расчёта, КП и заказа;
- защита от повторного создания заказа;
- отображение созданного заказа в карточке заявки.

## Безопасность

Выполнено:

- усилена защита пользовательских ролей;
- обычный пользователь не может самостоятельно повысить себе доступ;
- владелец сохраняет полный контроль;
- администратор может управлять сотрудниками в рамках своих прав;
- закрыт лишний публичный доступ к созданию заказа;
- убрана старая чрезмерно широкая политика доступа к потребностям;
- начат отдельный контур улучшений после аудита сайта и CRM.

## Ближайшие задачи

1. Усилить публичную форму: `request_id`, защита от повторного клика, server-side dedupe.
2. Подготовить миграцию для `leader_leads.request_id` и уникального индекса.
3. Перенести создание заказа из клиента в атомарную серверную операцию.
4. Завершить полный перенос CRM v4 из `lidercalculator` в `lider-bsk/crm/v4/`.
5. Внедрить мобильный навигатор этапов.
6. Проверить полный сценарий с телефона.
7. Сделать отдельную карточку заказа v4.
8. Добавить производство, монтаж, оплаты, расходы и фактическую прибыль.

## Ограничения

- CRM v4 пока не полностью перенесена в `lider-bsk`.
- Рабочая версия CRM v4 остаётся в `lidercalculator` до завершения проверки.
- Старую CRM v2 не удалять до завершения переноса и проверки.
- Таблицы `nav_*` не использовать для задач РА «Лидер».
- Удаления данных, таблиц, политик и функций выполнять только после отдельного подтверждения владельца.
