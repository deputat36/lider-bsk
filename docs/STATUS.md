## Публичный сайт

Выполнено:

- `robots.txt` существует и оформлен построчно;
- `sitemap.xml` существует и имеет XML-структуру;
- восстановлено видимое отображение логотипа в шапке сайта;
- добавлены CSS-ограничения от горизонтального переполнения;
- улучшена адаптация шапки, навигации, сеток и блоков под ноутбуки и планшеты;
- публичная форма передаёт `request_id`, `page_path`, `submitted_at` и технические данные источника;
- повторный клик по отправке блокируется на клиенте до завершения запроса.

## Supabase и публичные заявки

Проверено через Supabase:

- `leader-public-lead` активна, версия 6, публичная, без JWT;
- `leader-crm-leads` активна, версия 7, JWT включён;
- `leader-crm-orders` активна, версия 2, JWT включён.

Выполнено:

- в `leader_leads` добавлены `request_id`, `phone_normalized`, `source_page_path`, `submitted_at`, `client_user_agent`;
- добавлена уникальность `leader_leads.request_id`;
- добавлен индекс для поиска дублей по `phone_normalized` и дате создания;
- отправка заявки использует `on_conflict=request_id` и игнорирует повторную вставку дубля;
- создана таблица `leader_public_lead_audit`;
- RLS для `leader_public_lead_audit` включён;
- добавлены индексы аудита по дате, `request_id` и нормализованному телефону;
- чтение аудита доступно активным сотрудникам ролей `owner`, `admin`, `manager`;
- запись аудита разрешена публичному anon-контуру Edge Function;
- `leader-public-lead` пишет аудит для событий `accepted`, `suspicious`, `rejected`, `error`;
- ошибка записи аудита не блокирует получение основной заявки.

## CRM v4

Рабочий путь CRM v4 во временном репозитории:

1. Вход.
2. Список заявок.
3. Карточка заявки.
4. Потребности клиента.
5. Сохранённые расчёты.
6. Коммерческие предложения.
7. Создание заказа из согласованного КП.
8. Контроль заказов, производства, монтажа и финансов.
9. Управленческий дашборд.
10. Аудит публичных заявок.

Выполнено во временной CRM:

- создание заказа из согласованного КП переведено на серверное действие `create_order_from_offer`;
- карточка заявки стала более логичной: сводка, контакт и статус, потребности, расчёты, КП, заказ;
- технические данные формы сворачиваются;
- список заявок получил панель внимания: просрочено, без следующего шага, без телефона, новые больше суток;
- раздел `Финансы` открывается через штатный lazy-loader;
- раздел `Дашборд` открывается через `control-sections-lazy-loader-v1.js` из `management-dashboard-v3.js`;
- старый wrapper `management-dashboard-v2.js` отключён, чтобы не было дублей;
- добавлен `crm-ui-selfcheck-v1.js` для проверки загруженных разделов CRM;
- self-check теперь проверяет и `public_lead_audit`;
- добавлен модуль `public-lead-audit-v1.js`;
- раздел `Аудит заявок` читает `leader_public_lead_audit`, показывает последние 80 событий, счётчики и фильтры;
- `Аудит заявок` подключён через `site-cache-note-v1.js`.

Перенос начат в каталог:

`crm/v4/`

В основной репозиторий уже перенесены:

- `crm/v4/assets/v4/auth.js`;
- `crm/v4/assets/v4/functions-client.js`;
- `crm/v4/assets/v4/crm-v4-tabs-lite.js`;
- `crm/v4/assets/v4/site-cache-note-v1.js`;
- `crm/v4/assets/v4/crm-ui-selfcheck-v1.js`;
- `crm/v4/assets/v4/public-lead-audit-v1.js`.

`crm/v4/index.html` обновлён:

- добавлена базовая панель вкладок;
- подключён лёгкий обработчик вкладок;
- подключён загрузчик `site-cache-note-v1.js`;
- временно отображаются только реально перенесённые рабочие разделы: `Заявки` и `Аудит заявок`.

Важно:

- рабочая CRM v4 пока остаётся в `lidercalculator`;
- крупный `calculations.js` не переносился через веб-инструмент, потому что ответ GitHub по нему может обрезаться и есть риск создать неполный файл;
- крупные JS-модули лучше переносить локальным commit-пакетом или небольшими логическими пакетами.

## Коммерческие предложения и заказы

Выполнено:

- КП создаётся из сохранённого расчёта;
- клиентский текст КП не раскрывает внутренние финансовые данные;
- печатный/PDF-модуль КП переведён на встроенный HTML/CSS-логотип без внешней картинки;
- заказ создаётся только из согласованного КП;
- переносится клиент, позиции и суммы;
- связываются заявка, потребность, расчёт, КП и заказ;
- есть защита от повторного создания заказа.

## Автоматические проверки

GitHub Actions проверяет:

- `robots.txt`;
- XML-структуру `sitemap.xml`;
- что URL из `sitemap.xml` ведут на существующие файлы;
- наличие `assets/public-lead-form.css` и `assets/public-lead-form.js`;
- что форма сайта содержит `request_id`, `page_path`, `submitted_at`;
- что CSS формы содержит защиту от горизонтального переполнения;
- что CSS формы повторно не скрывает логотип сайта;
- базовый поиск случайно закоммиченных секретных ключей.
