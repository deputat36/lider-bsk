# Проверка аудита публичной заявки v8

Дата: 2026-06-25.

Использовать вместе с `docs/CRM_V4_TESTER_CHECKLIST.md`.

## Проверка новой заявки

1. Открыть `https://www.lider-bsk.ru/request.html`.
2. Нажать Ctrl + F5.
3. Отправить заявку с пометкой `Тест CRM v4 audit v8`.
4. Скопировать номер обращения из сообщения сайта.
5. Открыть основной CRM-контур.
6. Открыть раздел `Аудит заявок`.
7. Нажать `Обновить аудит`.
8. Найти событие по `request_id`.
9. Проверить статус `Принято`.
10. Нажать `Скопировать request_id`.
11. Вставить номер обращения в поле `Проверить request_id`.
12. Проверить, что виджет показывает `Цепочка полная`.
13. Сопоставить заявку и audit-событие по `request_id`.

## Проверка дубля

Повторная отправка с тем же `request_id` должна создать audit-событие `Дубль`, а не `Принято`.

## Что считать ошибкой

- номер обращения не показывается на сайте;
- событие не находится в `Аудит заявок`;
- нет кнопки `Скопировать request_id`;
- нет поля `Проверить request_id`;
- виджет не показывает `Цепочка полная` для корректной новой заявки;
- повторный `request_id` отображается как `Принято`, а не `Дубль`;
- заявка есть, но audit-события нет;
- audit-событие есть, но нет заявки, кроме ожидаемых `suspicious` или `rejected`.
