# Проверка аудита публичной заявки v8

Дата: 2026-06-29.

Использовать вместе с `docs/CRM_V4_TESTER_CHECKLIST.md`.

## Проверка новой заявки

1. Открыть `https://www.lider-bsk.ru/request.html`.
2. Нажать Ctrl + F5.
3. Отправить заявку с пометкой `Тест CRM v4 audit v8`.
4. Скопировать номер обращения из сообщения сайта.
5. Открыть основной CRM-контур.
6. Открыть раздел `Аудит заявок`.
7. Нажать `Обновить аудит`.
8. Найти событие по `request_id`.
9. Проверить статус `Принято`.
10. Проверить верхнюю сводку: `С request_id` должно быть больше 0.
11. Проверить верхнюю сводку: `Без request_id` не должно расти на корректных отправках формы.
12. Нажать `Проверить цепочку` в карточке audit-события.
13. Проверить, что виджет показывает `Цепочка полная`.
14. Нажать `Открыть заявку`, если кнопка появилась в виджете трассировки.
15. Сопоставить заявку и audit-событие по `request_id`.
16. Дополнительно проверить резервный сценарий: нажать `Скопировать request_id`, вставить номер обращения в поле `Проверить request_id` и получить тот же результат.

## Проверка дубля

Повторная отправка с тем же `request_id` должна создать audit-событие `Дубль`, а не `Принято`.

После дубля нужно проверить:

- в списке аудита появилась карточка `Дубль`;
- кнопка `Проверить цепочку` открывает трассировку по этому `request_id`;
- сводка `С request_id` учитывает это событие;
- сводка `Без request_id` не увеличилась из-за дубля.

## Что считать ошибкой

- номер обращения не показывается на сайте;
- событие не находится в `Аудит заявок`;
- нет сводки `С request_id` / `Без request_id`;
- корректная отправка попадает в `Без request_id`;
- нет кнопки `Проверить цепочку`;
- нет кнопки `Скопировать request_id`;
- нет поля `Проверить request_id`;
- виджет не показывает `Цепочка полная` для корректной новой заявки;
- повторный `request_id` отображается как `Принято`, а не `Дубль`;
- заявка есть, но audit-события нет;
- audit-событие есть, но нет заявки, кроме ожидаемых `suspicious` или `rejected`.
