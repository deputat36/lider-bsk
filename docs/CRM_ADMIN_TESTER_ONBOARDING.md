## Правила безопасного тестирования

Разрешено:

- открывать разделы CRM;
- обновлять списки;
- открывать вкладку `Доступ`;
- создавать invite только для явно тестового сотрудника и только с подтверждения владельца;
- закрывать только явно тестовый invite;
- создавать тестовую заявку вручную;
- добавлять тестовый комментарий к заявке;
- создавать тестовую потребность;
- создавать тестовый расчёт;
- создавать тестовое КП;
- создавать заказ только из явно тестового КП;
- проверять аудит заявок;
- проверять производство, монтаж, финансы и контроль заказов на чтение;
- фиксировать ошибки скриншотом и коротким описанием;
- создавать GitHub issue через шаблон `CRM v4 browser test`, если ошибка воспроизводится.

Не делать без отдельного подтверждения:

- удалять реальные данные;
- менять роли реальных пользователей;
- отключать реальные профили;
- закрывать реальные приглашения;
- менять настройки Supabase;
- менять реальные заказы клиентов;
- менять реальные финансовые данные;
- массово менять статусы заявок или заказов;
- отправлять клиентам реальные КП из тестовых сценариев.

Снятие доступа выполняется через вкладку `Доступ` или emergency SQL владельца проекта. Контрольный маркер отключения профиля: `leader_user_profiles.is_active = false`.

## Минимальный сценарий обучения

1. Открыть основную CRM v4.
2. Нажать Ctrl + F5.
3. Войти под `kvmbsk@yandex.ru`.
4. Открыть блок `Проверка загруженных разделов и доступа CRM`.
5. Проверить блок `Доступ`:
   - email: `kvmbsk@yandex.ru`;
   - роль: `admin`;
   - профиль: `Активен`;
   - статус входа без ошибки.
6. Проверить build marker `CRM build: 20260627-access-route-1`.
7. Открыть `Доступ` одним из способов:
   - прямой URL `https://deputat36.github.io/lider-bsk/crm/v4/?tab=user_admin`;
   - кнопка `Открыть доступ CRM` в карточке `CRM готова`;
   - верхняя вкладка `Доступ`.
8. Проверить, что раздел `Доступ к CRM` открылся.
9. Проверить, что видны пользователи CRM и приглашения.
10. Проверить блок `Разделы`: все строки должны быть `OK`, без `нет кнопки` и без `дублей`.
11. Проверить, что в самопроверке есть ссылка `Создать GitHub issue CRM v4 browser test`.
12. Проверить, что открывается меню CRM.
13. Открыть `Дашборд`.
14. Открыть `Заявки`.
15. Открыть любую безопасную тестовую заявку или создать новую тестовую заявку.
16. Добавить комментарий в историю заявки.
17. Создать тестовую потребность.
18. Создать тестовый расчёт.
19. Создать тестовое КП.
20. Открыть карточку КП.
21. Проверить печать/PDF КП.
22. Согласовать тестовое КП.
23. Создать заказ из тестового КП.
24. Открыть `Заказы` и найти созданный заказ.
25. Открыть `Контроль заказов`.
26. Открыть `Финансы`.
27. Открыть `Производство`.
28. Открыть `Контроль контактов`.
29. Открыть `Аудит заявок`.
30. Нажать `Проверить CRM` в диагностическом блоке.
31. Заполнить отчёт по шаблону `docs/CRM_V4_BROWSER_TEST_REPORT.md` или создать issue через `CRM v4 browser test`, если есть ошибка.
32. Если issue создан, указать в нём, заполнен ли `docs/CRM_V4_BROWSER_TEST_REPORT.md`, и ссылку или место заполненного отчёта.

## Что фиксировать как ошибку

Фиксировать проблему, если:

- вход не работает;
- CRM зависает после входа;
- самодиагностика не появляется после Ctrl + F5;
- в самодиагностике неправильный email;
- в самодиагностике роль не `admin`;
- в самодиагностике профиль не `Активен`;
- в самодиагностике есть `нет кнопки` или `дублей`;
- в самодиагностике нет ссылки `Создать GitHub issue CRM v4 browser test`;
- `CRM build: 20260627-access-route-1` не виден после Ctrl + F5;
- вкладка `Доступ` и кнопка `Открыть доступ CRM` не видны;
- прямой URL `?tab=user_admin` не открывает `Доступ к CRM`;
- для admin не видны пользователи CRM и приглашения;
- раздел меню не открывается;
- кнопка в меню дублируется;
- данные не загружаются;
- появляется ошибка Supabase;
- в консоли браузера есть 404 по файлам `assets/v4/`;
- кнопка нажимается, но ничего не происходит;
- карточка заявки или заказа не открывается;
- созданный тестовый объект не появляется после обновления;
- PDF КП не открывается;
- в клиентском тексте КП видны себестоимость, прибыль, маржа или цены подрядчиков;
- печатный лист производства или монтажа показывает лишние персональные данные клиента;
- аудит заявок не показывает тестовую отправку с сайта.
