# Улучшения по аудиту сайта и CRM РА «Лидер»

Дата: 2026-06-21.

## Что уже проверено

- `robots.txt` в актуальном репозитории уже оформлен построчно и указывает на основной sitemap.
- `sitemap.xml` уже имеет XML-структуру и содержит основные страницы сайта.
- Основной репозиторий проекта — `deputat36/lider-bsk`.
- CRM v4 пока переносится из `deputat36/lidercalculator` в `crm/v4/`.

## Что внедряется первым этапом

1. Зафиксировать текущие выводы аудита в документации проекта.
2. Добавить автоматические проверки статических файлов сайта.
3. Усилить защиту публичной формы от случайных дублей на клиенте.
4. Подготовить миграции для серверной идемпотентности и атомарной конверсии лида в заказ без прямого применения к боевой базе.

## Приоритеты

### Срочно

- Проверка `robots.txt` и `sitemap.xml` в CI.
- Проверка отсутствия секретных ключей в репозитории.
- Клиентский `request_id` для заявок с сайта.
- Документация по дальнейшему безопасному внедрению серверной защиты.

### Следующий этап

- Реализация server-side dedupe в Edge Function `leader-public-lead`.
- Миграция `leader_leads.request_id` и уникального индекса.
- Перенос связки создания заказа из клиента в одну серверную операцию.
- Завершение переноса CRM v4 в `lider-bsk/crm/v4/`.

## Ограничения

- Таблицы `nav_*` не трогать.
- Изменения базы выполнять только миграциями.
- Не удалять рабочие таблицы, поля, политики, функции и данные без отдельного подтверждения владельца.
- Рабочую CRM v2 и временную CRM v4 не удалять до полного тестирования.
