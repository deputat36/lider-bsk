# Request page clarity update — 2026-06-28

## Что изменено

Страница `request.html` стала более структурированной:

- первый экран сразу показывает форму заявки и объясняет, что можно заказать;
- добавлен блок `Что будет дальше` с тремя шагами после отправки;
- добавлен блок `Перед отправкой` с понятным списком данных для расчёта;
- добавлен блок `После отправки` с объяснением номера обращения и дублей;
- сохранены текущие контракты публичной формы: `request_id`, honeypot `website`, цель `lead_sent`, порядок скриптов `public-lead-reference-v1.js` перед `public-lead-form.js`.

## Что не менялось

- Supabase Edge Function не деплоилась.
- Схема базы, RLS, grants и политики не менялись.
- Контракт отправки заявки в `leader-public-lead` не менялся.

## Ручная проверка после публикации

Открыть:

`https://www.lider-bsk.ru/request.html`

Проверить:

1. На первом экране видны заголовок `Оставить заявку`, объяснение задачи и форма.
2. Есть блоки `Что будет дальше`, `Перед отправкой`, `После отправки`.
3. При отправке тестовой заявки форма показывает `Номер обращения`.
4. На мобильном экране форма не вылезает за ширину страницы.
5. Кнопка телефона ведёт на `8 980 245-74-71`.

## Автопроверка

UX-маркеры страницы добавлены в существующий workflow:

`.github/workflows/public-site-audit-check.yml`

Он проверяет наличие блоков `Что будет дальше`, `Перед отправкой`, `После отправки`, маркер версии страницы и критичные маркеры публичной формы.
